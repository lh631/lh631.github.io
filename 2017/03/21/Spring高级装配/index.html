Spring高级装配

Spring高级装配

###Profile

  • @Profile注解

指定某个Bean属于哪一个profole。
Spring在确定哪个profile处于激活状态时，需要依赖两个独立的属性：

如果设置了spring.profile.active属性，它的值会用来确定激活哪个profile。
如果没有设置，spring将会查找spring.profile.default的值。
如果均没有设置，那就没有激活的profile。

###条件化bean

  • @Conditional注解

条件化配置
实现Condition类，重写matches()方法，matches方法会得到ConditionContext和AnnotatedTypeMeetadata对象
ConditionContext是一个接口： 

public interface ConditionContext
{
    BeanDefinitionRegistry getRegistry();
    ConfigurableListableFactory getBeanFactory();
    Environment getEnvironment();
    ResourceLoader getResourceLoader();
    ClassLoader getClassLoader();
}

###处理自动装配的奇异性

  • @Primary注解

    标示首选Bean。

  • @Qualifier注解

    使用限定符的主要方式。

###Bean的作用域
Spring定义了多种作用域，包括：

  • 单例(Singleton)：在整个应用中，之后创建bean的一个实例。
  • 原型(Prototype)：每次注入或者通过Spring应用上下文获取的时候，都会创建一个新的bean实例。
  • 会话(Session)：在Web应用中，为每个会话创建一个bean实例。
  • 请求(Request)：在Web应用中，为每个请求创建一个bean实例。
  • @Scope注解：
    配置作用域
